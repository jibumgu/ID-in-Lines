# ID-in-Lines

`ID-in-Lines`는 정해진 선들이 모여 하나의 개인적인 ID처럼 보이도록 만든 인터랙티브 웹입니다. 사용자는 검은 바 형태의 선을 추가하고, 각 선에 계획을 기록하고, 완료 여부를 체크하면서 선 내부의 게이지가 차오르는 모습을 볼 수 있습니다. 마지막에는 스캔 기능을 통해 이 선들의 집합이 바코드에서 영감을 받은 구조였다는 점을 드러냅니다.

이 프로젝트는 HTML, CSS, JavaScript Canvas만으로 제작한 정적 웹입니다.

## 프로젝트 개요

처음의 핵심 주제는 "순서대로, 정해진 색상 카드들을 보여준다"였습니다. 이 주제를 리메이크하면서 단순히 카드를 넘기는 방식이 아니라, 현재의 계획이 미래의 나에게 어떤 형태의 자산처럼 남을 수 있는지를 표현하는 방향으로 확장했습니다.

작업의 관점은 다음과 같습니다.

- 현재 시점에서 미래를 바라본다.
- 계획은 지금 작성되지만, 실제 가치는 미래의 행동을 통해 생긴다.
- 정해진 순서와 위치를 가진 선들은 개인의 시간, 선택, 계획의 축적을 상징한다.
- 완성된 결과는 하나의 제품 바코드처럼 보이지만, 실제로는 사용자의 계획으로 구성된 개인적인 ID에 가깝다.

## 주요 기능

- 선 추가
- 선 이름 수정
- 선 위치 변경
- 선 굵기와 높이 조절
- 선 색상 변경
- 선 초기화 및 삭제
- 계획 추가
- Enter 키로 계획 추가
- 계획 수정 및 삭제
- 계획 완료 체크
- 계획 순서 변경
- 완료 비율에 따라 선 내부 게이지 상승
- 게이지에 잔잔한 물결 표현
- 스캔 버튼을 눌렀을 때 완료율 표시
- Canvas 화면 확대 및 축소
- 마우스 드래그로 Canvas 이동
- 마우스 위치 기준 확대 및 축소
- 로컬 스토리지 저장
- 새로고침 후에도 데이터 유지
- 모바일 반응형 컨트롤러 스크롤
- 키보드 접근성 일부 지원

## 사용 방법

1. `선 추가` 버튼을 눌러 바코드의 선을 추가합니다.
2. 선을 선택한 뒤 이름, 굵기, 높이, 색상을 조정합니다.
3. `계획 추가` 영역에 계획을 입력합니다.
4. Enter 키 또는 `계획 추가` 버튼으로 계획을 추가합니다.
5. 계획을 완료하면 체크박스를 선택합니다.
6. 완료된 계획 수에 따라 선택한 선 내부의 게이지가 차오릅니다.
7. `스캔` 버튼을 누르면 각 선 아래에 완료 비율이 표시되고, 전체 구조가 바코드처럼 인식됩니다.

## 조작 방식

Canvas 영역에서는 다음 조작을 사용할 수 있습니다.

- 마우스 드래그: 화면 이동
- 마우스 휠: 확대 및 축소
- `+`: 확대
- `-`: 축소
- `0` 또는 `Home`: 보기 초기화
- 방향키: 화면 이동
- `Escape`: 선택 해제

## 저장 방식

데이터는 서버로 전송되지 않고 브라우저의 `localStorage`에 저장됩니다.

저장되는 정보는 다음과 같습니다.

- 선 목록
- 선 이름
- 선 크기와 위치
- 선 색상
- 계획 목록
- 계획 완료 여부
- 스캔 표시 상태

브라우저 데이터 삭제, 시크릿 모드 사용, 저장 제한이 있는 환경에서는 데이터가 유지되지 않을 수 있습니다.

## 반응형 구조

데스크톱에서는 Canvas 작업 영역과 오른쪽 컨트롤러가 나란히 배치됩니다.

모바일에서는 Canvas가 위쪽에 배치되고, 컨트롤러가 아래쪽에 배치됩니다. 모바일 화면에서는 컨트롤러 영역 자체가 독립적으로 스크롤되도록 구성했습니다. 이렇게 하면 Canvas 화면은 유지하면서 선과 계획 설정을 아래 컨트롤러에서 계속 조작할 수 있습니다.

## 파일 구조

```text
ID-in-Lines/
├── index.html
├── css/
│   └── style.css
├── js/
│   └── index.js
├── README.md
└── .gitignore
```

## 실행 방법

정적 웹이므로 별도의 빌드 과정이 필요하지 않습니다.

가장 간단한 방법은 `index.html` 파일을 브라우저에서 여는 것입니다.

로컬 서버로 확인하려면 프로젝트 폴더에서 아래 명령을 실행할 수 있습니다.

```bash
python -m http.server 4173 --bind 127.0.0.1
```

그 다음 브라우저에서 아래 주소를 엽니다.

```text
http://127.0.0.1:4173/
```

## 디자인 방향

이 프로젝트는 바코드라는 소재를 처음부터 직접 설명하지 않는 방식으로 설계되었습니다. 사용자가 먼저 "선", "순서", "계획"을 경험한 뒤, 스캔이라는 동작을 통해 바코드적 구조를 나중에 인식하도록 만들었습니다.

색상은 선 자체에만 직접 적용할 수 있고, 게이지는 선택한 색상을 기반으로 파스텔톤으로 변환됩니다. 이는 원본 선의 강한 인상과 계획 진행률의 부드러운 변화를 분리하기 위한 선택입니다.

## 접근성 메모

현재 버전에서는 다음 항목을 고려했습니다.

- Canvas에 키보드 포커스 제공
- Canvas 조작에 대한 보조 설명 제공
- 선택된 선과 계획 개수에 `aria-live` 상태 제공
- 계획 목록에 `list`와 `listitem` 의미 제공
- 모바일 터치 영역 확대
- 고대비 모드 일부 대응

Canvas 기반 인터랙션 특성상 모든 기능이 완전한 접근성을 갖춘 것은 아니며, 이후 개선 여지가 있습니다.

## 앞으로 개선할 수 있는 점

- 데이터 내보내기 및 가져오기
- 여러 바코드 작품을 저장하고 전환하는 기능
- 계획 카테고리 또는 태그
- 완료율 히스토리
- 더 정교한 모바일 터치 제스처
- Canvas 요소의 접근성 보완
- 사용자가 직접 예시 데이터를 초기화하거나 불러오는 기능

## 영감에 대한 태도

이 프로젝트는 바코드의 시각 구조에서 영감을 받았지만, 실제 상품 바코드를 생성하거나 표준 바코드 규격을 구현하는 프로젝트는 아닙니다. 바코드의 "선들이 모여 하나의 값을 만든다"는 개념을 개인의 계획과 연결해 재해석한 인터랙티브 웹입니다.
